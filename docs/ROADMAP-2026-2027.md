## 技術実現性の評価

### 月1〜2プロダクトを継続できる条件

| 条件 | 現状 | 必要なアクション |
|---|---|---|
| **AIコーディング活用** | Claude Code導入済み | MVPの70%をAI生成→人間がレビューする体制 |
| **共通技術基盤** | Next.js + Prisma + Gemini + Neon | 新プロダクトは同スタックで横展開 |
| **マイクロアーキテクチャ** | 各リポジトリ独立 | 共通コンポーネントをnpmパッケージ化（Phase 2） |
| **データ基盤** | 卸業者DB（3,567商品）+ OCR | 新プロダクトの9割がこのデータを活用 |

### リスクと対策

| リスク | 深刻度 | 対策 |
|---|---|---|
| 品質 vs 速度のトレードオフ | 高 | タイプ別に定義（Full/Mini/SEO Tool）。SEO Toolは3日MVPを許容 |
| SEOは即効しない（3〜6ヶ月） | 中 | 既存顧客2,500店を最初のユーザーベースに。SEOは5月から仕込む |
| 10万ユーザーは無広告だと挑戦的 | 高 | 1本バズれば現実的。食材相場チェッカーが最有力。PRも重要 |
| エンジニアリングキャパシティ | 高 | Simple productから着手。Full Productは月1本、SEO Toolは別枠 |
